Below is the api documentation
Method Route Parameters Access Description
POST /api/auth/register body : JSON{name, email, password} Public A route for user to register
POST /api/auth/login body : JSON{email, password} Public A route for user to login
GET /api/profile headers : Authorization : Bearer token Private A route for user to get his/her profile
PUT /api/profile/update
  • headers : Authorization : Bearer token
  • body : JSON{name}
    		•  Private A route for user to update his/her profile
    POST /api/recovery/sendOTP body : JSON{email} Public A route for user to get OTP for reseting password
    PUT /api/recovery/reset body : JSON{id,password} Public A route for user to reset password
    POST /api/profile/upload
  • headers : Authorization : Bearer token
  • body : JSON{url}
    		•  Private A route for user to upload/change profile pic
    GET /api/admin/users headers : Authorization : Bearer token Admin A route for admin to get all users
    DELETE /api/admin/delete/:id
  • headers : Authorization : Bearer token
  • param : {id}
    		•  Admin A route for admin to delete a user
    PUT /api/admin/update
  • headers : Authorization : Bearer token
  • body : JSON{id, isAdmin}
    		•  Admin A route for admin to update a user's admin rights
    POST /api/contact body : JSON{name, email, message} Public A route for user to make contact with the admin or project team
